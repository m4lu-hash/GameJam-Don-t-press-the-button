# Game Design Document — "别按那个键" (Don't Press That Key)

> **版本**: 0.1 — 初稿  
> **引擎**: Godot 4.7  
> **类型**: 2D 俯视角 塔防基础上贴合按钮点击机制
> **主题**: 别按那个键

---

## 1. 游戏概述

### 1.1 一段话简介

"别按那个键"是一款桌面隐喻风格的塔防游戏。敌人化身为鼠标光标，试图点击地图上的"按钮"——你必须部署钓鱼弹窗、炮塔和区域打击等各式各样的手段，在它们触发"失败按钮"前将其消灭。同时地图上还散落着其他类型的按钮，他们会带来各种各样的效果：负面效果、增益效果和随机效果，以及还有一些短期收益长期减益甚至负面的按钮，玩家需要在这些按钮的诱惑和威胁之间做出权衡，合理分配资源，才能成功抵御敌人的进攻，守护"失败按钮"。（这些按钮全都可以被玩家触发，部分可以被敌人触发）

### 1.2 平台与规格

| 项 | 值 |
|---|---|
| 平台 | PC (Windows) 后期考虑手机移植 |
| 输入 | 纯鼠标操作 |
| 分辨率 | 1280P |
| 视角 | 2D 俯视 |

## 2. 核心游戏循环

### 2.1 游玩流程

准备阶段（观察地图的布局和各个按钮的文本提示等，同时部署防御） -> 第x波来袭-战斗阶段（游戏开始生成敌人，玩家在这个阶段可以部署防御和撤回防御） -> 第x波结束-调整阶段（如果有随机按钮或者一些延迟效果将会触发，玩家在这个阶段可以部署防御和撤回防御） -> 继续战斗然后调整直到游戏结束（胜利或失败） -> 结算阶段（显示统计数据和奖励，让玩家能够带到下一关）

#### A. 准备阶段
- 持续 30 秒（可配置）
- 商店面板滑入，然后玩家可以购买/升级防御，也可以折叠起来
- 部署的时候商店面板折叠，玩家可以直接点击地图放置防御（如果有足够碎片），放置完成后商店面板再次滑入
- 玩家使用**代码碎片**购买/升级防御
- 新的Debuff按钮可能在此时出现在地图上
- 点击"Ready"可提前开始下一波（奖励额外碎片）

#### B. 战斗阶段（波次进行中）
- 敌人从地图边缘的生成点出现
- 敌人寻路走向随机按钮
- 玩家可在战斗中放置防御设施
- 炮塔自动射击（如果有）
- 所有敌人被消灭后波次结束

#### C. 调整阶段（波次之间）
- 玩家可以在此阶段调整防御布局
- 如果有随机按钮或者一些延迟效果将会触发
  
#### D. 结算阶段（游戏结束）
- 显示统计数据和奖励，让玩家能够带到下一关

### 2.2 胜负条件

| 结果 | 条件 |
|---|---|
| **胜利** | 存活 20 波 |
| **失败** | 失败按钮被点击满 **3 次**（"系统稳定性"= 3条命） |

## 3. 游戏对象详细设计

### 3.1 按钮系统

#### 3.1.1 失败按钮
| 属性 | 值 |
|---|---|
| **外观** | 大型红色按钮，标注"DO NOT PRESS" |
| **位置** | 地图中央，固定不动 |
| **生命值** | 3 次点击 = 游戏结束 |
| **无敌时间** | 每次被点击后有 2 秒无敌（防止连点器一秒清空3条命） |
| **视觉反馈** | 每次被点击 → 按钮裂痕加深、屏幕抖动、红色闪烁、警报音效 |

#### 3.1.2 Debuff按钮
| 属性 | 值 |
|---|---|
| **数量** | 每波 3-4 个（随机子集出现） |
| **位置** | 每波随机位置（不与已有防御重叠） |
| **点击次数** | 1 次（老实人一次即触发）/ 连点器一次访问可点击 3 次 |
| **触发后** | 按钮消失，效果在结算阶段生效 |

#### 3.1.3 随机按钮（优先级2，可选）
| 属性 | 值 |
|---|---|
| **出现频率** | 每 3-4 波出现一次 |
| **效果** | 50% 奖励20碎片 / 50% 随机惩罚 |
| **外观** | 黄色/金色，带 "?" 图标 |

#### 3.1.4 Debuff效果表

| 效果名称 | 惩罚内容 | 持续时间 | 视觉提示 |
|---|---|---|---|
| **系统卡顿** | 所有炮塔射击频率 -40% | 1 波 | 屏幕间歇卡顿效果 |
| **屏幕变暗** | 视野范围 -50% | 30 秒 | 暗角遮罩 |
| **敌人超频** | 所有敌人移动速度 +30% | 1 波 | 敌人散发红光 |
| **紧急生成** | 立即额外生成 3-5 个老实人 | 即时 | 闪烁警告 |
| **系统更新** | 下一只飙车党变为移动堡垒（免疫伤害） | 直到触发 | 飙车党金色光芒 |

---

### 3.2 防御系统

#### 3.2.1 钓鱼窗口
| 属性 | 值 |
|---|---|
| **放置消耗** | 30 碎片 |
| **放置方式** | 点击空地（不可放在路径上、按钮上） |
| **使用次数** | 3 次（升级到 5 次，消耗 50 碎片） |
| **引诱半径** | 150 像素 |
| **引诱规则** | 范围内的敌人重新寻路到窗口。<br>连点器 50% 抵抗，坦克完全免疫 |
| **外观** | 仿Windows XP错误弹窗，带诱人的"点此修复"按钮 |

#### 3.2.2 射击炮塔
| 属性 | 值 |
|---|---|
| **放置消耗** | 40 碎片 |
| **射程** | 200 像素 |
| **射击频率** | 1.5 秒/发（可升级到 1.0 秒，50 碎片） |
| **伤害** | 1 击=老实人/飙车党死亡<br>2 击=连点器死亡<br>5 击=坦克死亡 |
| **瞄准模式** | 默认"最近敌人"，可升级切换为"最强"或"最快"（30碎片） |
| **弹丸** | 小型能量弹，400 px/s，轻微追踪 |
| **外观** | 小天线/卫星锅，旋转瞄准，发射能量弹 |

#### 3.2.3 盾牌猛击
| 属性 | 值 |
|---|---|
| **激活方式** | 按空格键 → 点击目标区域 |
| **冷却时间** | 35 秒（可升级到 25 秒，80 碎片） |
| **AOE半径** | 120 像素 |
| **效果** | 瞬杀范围内所有敌人（包括坦克） |
| **限制** | 技能型，不可额外放置（全局 1 个） |
| **外观** | 巨型盾牌/锤子从天而降，砸出坑痕 |

---

### 3.3 敌人系统

#### 3.3.1 属性总表

| 类型 | 昵称 | HP | 移速(px/s) | 点击次数 | 钓鱼抗性 | 特殊能力 |
|---|---|---|---|---|---|---|
| 老实人 | Honest | 1 | 80 | 1 | 0%（必定被引诱） | 无 |
| 连点器 | AutoClicker | 2 | 70 | 3 | 50%（随机抵抗） | 一次访问点击3次 |
| 飙车党 | Speedster | 1 | 180 | 1 | 0% | 转向半径大，容易冲过头 |
| 坦克 | Tank | 5 | 40 | 1 | 100%（免疫引诱） | 高血量 |

#### 3.3.2 敌人AI状态机

```
[生成] → [选择目标] → [移动至目标]
                            ↓
                   [在引诱范围内？] → 是 → [改向钓鱼窗口]
                            ↓ 否
                   [到达目标？] → 是 → [点击按钮] → [死亡]
                            ↓
                   [被击杀？] → 是 → [死亡动画] → [释放节点]
```

**详细行为**：
1. **选择目标**：随机选取一个按钮（权重：40%最近按钮、20%失败按钮、40%随机Debuff按钮）
2. **移动**：使用 NavigationAgent2D 寻路。飙车党每0.5秒加一个随机垂直偏移（画龙效果）
3. **引诱判定**：每0.3秒检测一次。若在钓鱼窗口引诱范围内且不免疫 → 改向窗口
4. **点击**：到达目标后停止，播放点击动画。每次点击0.5秒。连点器连击3次用时1.5秒
5. **死亡**：短暂粒子爆散动画 → 释放碎片

#### 3.3.3 精英敌人（Priority 2，波次10+）
| 属性 | 值 |
|---|---|
| **外观** | 比普通敌人大一号，头顶星标/皇冠 |
| **属性加成** | 2倍HP，1.2倍移速 |
| **死亡效果** | 掉落3倍碎片 + 标记最近按钮（Button Rush） |
| **出现率** | 波次10起每波1只，最多2只/波 |

---

### 3.4 敌人组合技

| 组合名 | 触发条件 | 效果 | 持续时间 |
|---|---|---|---|
| **移动堡垒** | 飙车党存活 + 系统更新Debuff激活 | 飙车党变为免疫伤害，保持高速 | 到飙车党死亡或波次结束 |
| **按钮集火** | 精英敌人死亡时标记一个按钮 | 全场所有普通敌人改向该按钮 | 5 秒 |

## 4. 资源经济

### 4.1 代码碎片获取

| 来源 | 收益 |
|---|---|
| 击杀老实人 | +5 |
| 击杀连点器 | +10 |
| 击杀飙车党 | +8 |
| 击杀坦克 | +15 |
| 击杀精英 | +30 |
| 玩家手动重置Debuff按钮（先于敌人点击） | +15 |
| 提前按"Ready"结束准备阶段 | 奖励剩余秒数 × 1 |

### 4.2 代码碎片消耗

| 项目 | 消耗 |
|---|---|
| 放置钓鱼窗口 | 30 |
| 放置射击炮塔 | 40 |
| 升级钓鱼窗口次数(3→5) | 50/个 |
| 升级炮塔射速(1.5s→1.0s) | 50/个 |
| 升级炮塔瞄准模式 | 30/个 |
| 缩短盾牌猛击冷却(35s→25s) | 80 |

### 4.3 经济平衡目标
- 起始资金：50 碎片（足够1个炮塔或1个钓鱼窗口）
- 每波理想状态：放置 2 个新防御 或 升级 1-2 个已有防御
- 第20波理想状态：6-8个炮塔 + 3-4个钓鱼窗口 + 全满级技能

## 5. UI/UX设计

### 5.1 HUD（战斗中常驻）
| 位置 | 元素 |
|---|---|
| 左上角 | 波次数 "Wave 5 / 20" |
| 顶部中央 | 碎片数量 + 图标 |
| 右上角 | 系统稳定性 ❤❤❤ （3颗心） |
| 底部中央 | 盾牌猛击冷却环 |
| 底部右侧 | 快捷购买按钮（战斗中放钓鱼窗口） |

### 5.2 商店面板（准备阶段）
- 从屏幕右侧滑入
- 分区：
  - "放置防御"（钓鱼窗口、炮塔，显示价格，拖放到地图预览）
  - "升级"（每个已有防御的升级选项，盾牌猛击冷却升级）
  - "Ready"按钮（大号绿色，居中底部）
- 显著显示当前碎片余额

### 5.3 按钮外观
| 按钮类型 | 大小 | 颜色 | 特征 |
|---|---|---|---|
| 失败按钮 | 80×80px | 红色 | 脉动发光，"DO NOT PRESS" |
| Debuff按钮 | 50×50px | 灰色+彩色边框 | 边框颜色对应效果类型 |
| 随机按钮 | 50×50px | 金色 | "?" 图标 |
| 全部按钮 | — | — | 带点击进度条（已点击数/总需点击数） |

### 5.4 波次通告
- "Wave X" 文字淡入淡出在屏幕中央
- Debuff触发 → 屏幕顶部红色横幅通知
- 组合技触发 → 橙色横幅警告

### 5.5 游戏结束画面
- 失败："SYSTEM CRASHED" + 蓝屏死机风格
- 胜利："SYSTEM SECURED" + 胜利风格
- 统计：波次存活、击杀数、碎片收入
- 按钮：重新开始 / 返回主菜单

## 6. 视觉风格

### 6.1 总体美学 — "桌面隐喻"
整个游戏发生在一个风格化的计算机桌面上。

### 6.2 各元素视觉
| 元素 | 描述 |
|---|---|
| **地图/背景** | Windows风格桌面壁纸，底部任务栏（纯装饰），桌面上散落图标（障碍物） |
| **网格** | 半透明网格覆盖，辅助防御放置定位 |
| **老实人** | 标准白色鼠标光标（箭头） |
| **连点器** | 红色光标 + 点击动画（光标尖端上下敲击） |
| **飙车党** | 蓝色光标 + 拖尾运动线 |
| **坦克** | 齿轮/设置图标光标，体积更大 |
| **钓鱼窗口** | Windows 2000 风格的错误对话框，灰色+红X |
| **射击炮塔** | Wi-Fi/信号图标，旋转锁定目标 |
| **盾牌猛击** | 绿色巨型锤子/盾牌砸下，留下短暂坑痕贴花 |
| **敌人死亡** | 像素粒子爆散（破碎的光标） |
| **按钮被点击** | 按钮下压动画 + 涟漪扩散 |
| **碎片拾取** | 绿色代码括号 "{}" 飘向HUD |

## 7. 音效概要

| 事件 | 音效 |
|---|---|
| 炮塔射击 | 短促激光 "zap" |
| 敌人死亡 | 爆裂声 / 破碎声 |
| 按钮被敌人点击 | Windows 错误提示音 "beep" |
| 盾牌猛击 | 沉重撞击声 "thud" |
| 波次开始 | 上升音阶 |
| 游戏失败 | 蓝屏音效，系统崩溃 |
| 游戏胜利 | 胜利号角 |
| 碎片获得 | 金币/提示音 |
